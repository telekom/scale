## Fields Structure

The `fields` property is an array of field options. Each set of options defines the parameters for that field/column.

The field columns will display in the same order as this array.

The number of objects in the `fields` array determines the number of items that must be in each row supplied to the `rows` property.

The only required field option is `type`. All common options are listed further down the page, as well as individual cell type options.

```html
const fields = [ { type: 'number', label: 'ID', }, { type: 'text', label:
'Name', }, { type: 'date', label: 'Time', }, ];
```

## Rows Structure

The `rows` property is an array of row data. Each row is a flat array that must match the length of the `fields` property array.

Each item in the row contains the cell value for the associated field type, and must therefore match the expected format of the field options.

Eg. the first item in the fields example above is a `text` field type, therefore the first item in each row must be a `string`. Each type's expected format is documented further down the page.

```html
const rows = [ [1, 'John', '12:30'], [2, 'Mary', '2:12'], [3, 'Patek', '16:01'],
[4, 'Heidi', '3:15'], [5, 'Muhammad', '21:45'], ];
```

## Field Cell Types

- `actions`
- `checkbox`
- `date`
- `graph`
- `html`
- `link`
- `number`
- `select`
- `tags`
- `text`

## Field Options

##### Common options, and their defaults

- `type: string`
- `label?: string`
- `maxWidth?: number = Infinity`
- `minWidth?: number = 20`
- `mobileTitle?: boolean = false`
- `resizable?: boolean = true`
- `sortable?: boolean = false`
- `sortBy?: 'number' | 'text' = 'text'`
- `textAlign?: 'left' | 'center' | 'right' = 'left'`
- `stretchWeight?: 'number'`
- `visible?: boolean = true`
- `width?: number`
